## Breaking Changes Detail

### 20260321051704 - UpdatePaperEmbeddingVectorTo768

**Impact**: High - all embedding data is destroyed

**Description**: This migration changes the embedding vector dimension from 1536 to 768 to align with the Snowflake/snowflake-arctic-embed-m-v1.5 model. The migration includes `DELETE FROM paper_embeddings;` which removes all existing embedding data.

**Action Required**: After applying this migration, paper embeddings must be regenerated by re-running summarization or document processing jobs.

### 20260415014020 - AddPaperEmbeddingVectorHnswIndex

**Impact**: Medium - requires pgvector extension

**Description**: This migration adds the `document_chunks` table and a new FK column `document_chunk_id` on `paper_embeddings`. It also creates an HNSW index for vector similarity search.

**Prerequisites**:
```sql
CREATE EXTENSION IF NOT EXISTS pgvector;
```

## Schema Notes

| analysis_results | result_json | See `AnalysisResult.ResultJson` XML docs |
| trend_results | result_json | See `TrendResult.ResultJson` XML docs |

### Indexes

- `ix_paper_embeddings_vector_hnsw`: HNSW index for vector similarity (L2 distance)
- `IX_paper_embeddings_DocumentChunkId`: B-tree index for chunk-based queries
- `IX_document_chunks_PaperDocumentId_ChunkIndex`: Unique composite index

## Upgrading from Pre-v1.3

If upgrading from a schema prior to the vector dimension change:

1. Backup existing data if paper_embeddings content is needed
2. Apply migrations in order
3. Re-generate embeddings via API:
   - Resubmit papers for summarization
   - Re-process paper documents

## Rollback Notes

Most migrations can be rolled back via `dotnet ef database update PreviousMigration`. However:

- `20260321051704`: Rollback will restore vector to 1536 but data is already deleted
- `20260415014020`: Rollback will drop `document_chunks` table and `document_chunk_id` column
